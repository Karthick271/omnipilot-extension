# Contributing

## Commit Style
Use Conventional Commits:
- `feat:` new features
- `fix:` bug fixes
- `docs:` documentation
- `refactor:` code refactors
- `chore:` chores like deps/bump

## Branching
- `main` — stable
- `dev` — integration
- topic branches: `feat/...`, `fix/...`

## PR Checklist
- Keep PRs small
- Add/update docs
- Record changes in `CHANGELOG.md`
